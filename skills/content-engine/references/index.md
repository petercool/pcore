## 3. Execution Instructions

When given a topic, the agent must:

1. **Niche alignment check.** Read this file's Identity section. Does the topic fit "investor-grade judgment applied to AI, trading infrastructure, or market intelligence"? If not, reject with a specific reason and stop.
2. **Load voice.** Read [[brand-voice]]. Internalize the four voice principles (charisma, respect, trust, never-talk-down) and the anti-pattern list. These are gates, not suggestions.
3. **Select hooks.** Read [[hooks]]. For each active platform, pick 2–3 candidate hooks from the approved taxonomy. Run the respect test on each. Exclude any hook in the voice-violations list.
4. **Execute the repurposing chain** per [[repurpose]]:
   - [[x]] first — forces brevity, finds the core.
   - [[linkedin]] second — add narrative, institutional vocabulary, different angle.
   - [[newsletter]] third — deepest, most personal, include "what could invalidate this view."
5. **Adapt voice per platform.** For each platform, read the matching section in [[platform-tone]] and the platform node ([[linkedin]] / [[x]] / [[newsletter]]). The voice stays the same — the energy and format change.
6. **Apply the annoyance test** (from [[repurpose]]). Would a reader following all three platforms feel the content is repeated? If yes, return to step 4 and rethink the angle, not just the wording.
7. **Stub outputs** for the 5 inactive platforms — a one-line placeholder noting the stub file is empty and will be filled when the platform goes active.
8. **Apply scheduling** per [[scheduling]] — note the recommended posting day/time per platform alongside each draft.

## 4. Rethink ≠ Reformat (Critical Rule)

The output is NOT 3 copies of the same text reformatted for each platform. It is 3 pieces that each *think about the topic differently*:

- Different **angle** (which facet of the topic the post opens with).
- Different **hook** (the opening 1–3 lines — chosen from [[hooks]] per platform fit).
- Different **voice** energy (per [[platform-tone]]).
- Different **structure** and **format** (per platform file and [[content-types]]).
- Different **depth** (X = surface insight, LinkedIn = walk-through, Newsletter = full reasoning + what could invalidate).

The reader following Peter everywhere should experience three distinct value-adds — not one message delivered thrice.

## 5. Iteration

Content engines compound. After each weekly batch:
- Which hook types drove engagement? Add winners to [[hooks]], retire losers.
- What topic clusters worked? Note them for next week's batch.
- Any voice calibration to add? Update [[platform-tone]].
- New audience signals (DMs, comments, replies)? Update [[builders]] or [[casual]].

Weak nodes = weak output. Strong, current nodes = compounding edge.
