# x.md — X (Twitter) Playbook

## Platform DNA

**"The charismatic investor voice — direct, grounded, never smug."**

X is where Peter posts most frequently. The voice is punchier than LinkedIn, more opinionated, willing to carry light humor and dry observation — but the rules from `brand-voice.md` still apply. No "hot takes" that substitute provocation for insight. No "most people get this wrong" framings. Peer voice, sharpened.

## Content Rules

- **Character limit:** 280 per tweet. Threads: 5–12 tweets, each standalone.
- **Sentence case** by default. Lowercase for emphasis occasionally is fine — but pure-lowercase isn't Peter's register.
- **Every line earns the next.** If a sentence doesn't pull the reader forward, cut it.
- **One idea per tweet.** Threads that blur ideas lose readers.
- **No "1/", "2/" thread counters.** Each tweet should work alone. If the reader only sees tweet #4, it should still be readable.
- **No cliffhangers** like "the reason will shock you…" — manipulative, and Peter's audience sees through it.
- **Hashtags:** rarely. One strategic hashtag per post is fine; more looks desperate.
- **Links:** fine in the main tweet for X. External links don't suppress reach on X the way they do on LinkedIn.
- **Quote-tweets:** use for adding analysis. Not for dunking.

## Voice Calibration

Per `voice/platform-tone.md` → X section:
- Direct, punchy, charismatic
- Light humor and dry asides welcome — sarcasm out
- Contrarian takes allowed only when framed with reasoning
- "Here's the non-obvious angle" beats "everyone's wrong"
- Sentence case default; lowercase as occasional tonal tool

## Five Formats That Work

### 1. Non-Obvious Take
A single sharp observation with a clear reasoning anchor. Not hot take — measured, specific.

**Hook options:** Non-Obvious Angle, Data Lead
**Example:** "NVDA beat by 32%. The capex read most analysts are pricing in is durability. The one they're not is energy sensitivity at the marginal cluster."

### 2. Thread (Step-by-Step or Framework)
Reasoning walked through in 5–12 tweets. Each tweet stands alone. Ends with a concrete takeaway or question.

**Hook options:** Framework, Walk-Through, Data Lead, Collision
**Example topic:** "The 4-step framework I use for sizing speculative positions"

### 3. Proof Post
Before/after with specific numbers and the mechanism. Short. High-credibility format.

**Hook options:** Proof
**Example:** "Before: 14-day research cycle. After: 3 days. Changed one thing — the data-loading stage. Here's the mechanism."

### 4. Resource Drop
A curated list with real context per item. "Saved for later" format. Must have a clear through-line.

**Hook options:** Value, Data Lead (as context opener)
**Example topic:** "5 papers that changed how I think about regime detection — with the specific insight from each"

### 5. Quote-Tweet with Analysis
Quote-tweet adds mechanism, context, or a sharp disagreement — never mockery. Attribution is a feature.

**Hook options:** varies — often Non-Obvious Angle or Collision
**Rule:** if the quote-tweet doesn't add specific analysis, don't post it. Retweet instead.

## Post Structure Templates

### Single tweet
```
[Hook — one sentence that also contains the thesis]

[Supporting line — specific number, case, or mechanism]

[Optional close — concrete takeaway or genuine question]
```
(All within 280 chars.)

### Thread (5–12 tweets)
```
Tweet 1: Hook (per hooks.md) + the core claim
Tweet 2: Context or first supporting point
Tweets 3–N: Walk-through or step-by-step
Second-to-last tweet: Counterargument or invalidation
Last tweet: Concrete takeaway or question (NOT "follow for more")
```

## Posting Strategy

- **Cadence:** 7–10 posts/week (per `engine/scheduling.md`)
- **Days:** Daily Mon–Fri; weekends lighter (lower institutional engagement)
- **Times:** 8–9am (pre-market read), 12–1pm (midday check), 5–6pm (post-close reflection)
- **Thread timing:** Tuesday 8am is optimal for long threads. Thursday 8am secondary.

## Audience

- **Primary:** builders (per `audience/builders.md`) — most of Peter's X audience
- **Secondary:** casual — some allocators and strategic professionals use X as a daily read

The audience skews builder-heavier on X than on LinkedIn. Technical depth is more welcome here; strategic translation less necessary.

## Repurposing Notes

X goes **first** in the chain per `engine/repurpose.md`. The reasoning:
- If the idea survives compression to 280 chars (or a tight thread), it has a core.
- If it can't, it's probably not a fully-formed insight yet.

X on a topic should differ from LinkedIn on the same topic in these ways:

| Dimension | X | LinkedIn |
|-----------|---|----------|
| Entry angle | Sharpest single facet, often the data or the collision | Different facet — often framework or walk-through |
| Voice energy | Punchy, sentence fragments welcome | Professional-warm, full sentences |
| Format | Thread or single tweet | Long post |

Never post the LinkedIn post and then "copy for X" — draft X first, LinkedIn second with a different angle.

## Things to Avoid on X

- "Hot take:" / "Unpopular opinion:" / "Controversial:" — smug framings
- "Thread 🧵" as the hook — weak; the hook should be the first substantive line
- Cliffhangers ("stay tuned for the real reason…")
- "Follow for more" as a close
- Engagement-bait ("like if you agree, RT if you disagree")
- Quote-tweeting to dunk
- Threads that pad to look longer

## X-Specific Opportunities

- **Quote-tweet analysis** — high leverage format. Peter can add specific context to someone else's post in a way that serves both his audience and the original poster.
- **Reply guy mode (selectively)** — thoughtful replies to respected accounts can build reach faster than standalone posts. Not mass-reply; selectively, with substance.
- **Image tweets** — charts, tables, or diagrams embedded in a tweet outperform pure text by 2–3x on this platform.
- **Polls** — occasionally useful for surfacing genuine audience questions. Don't use as a lazy post substitute.
