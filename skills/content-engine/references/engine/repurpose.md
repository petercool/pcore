# repurpose.md — The Repurposing Chain

> One topic in, platform-native posts out. The chain order is non-negotiable. The annoyance test is a hard gate.

## The Chain Order (v1 = 3 active platforms)

### Step 1 — X first
**Why first:** Forces brevity. Finds the core. If the idea can't survive compression to 280 characters (or a tight 5–12 tweet thread), it isn't ready for the longer platforms.

**Output:** one single tweet or one thread. Hook from `hooks.md`, voice per `platform-tone.md` → X.

### Step 2 — LinkedIn second
**Why second:** With the core locked in from X, LinkedIn adds narrative, institutional framing, and a different angle — not a reformatted X.

**Critical:** the LinkedIn angle should be different from the X angle. Same topic, different entry point. If X led with a data surprise, LinkedIn might lead with a framework or a walk-through.

**Output:** 1,300–2,000 character post, native LinkedIn format, fold ~210 chars. Hook per `hooks.md`, voice per `platform-tone.md` → LinkedIn.

### Step 3 — Newsletter third
**Why third:** With X and LinkedIn drafted, the Newsletter goes deeper. Full reasoning, "what could invalidate this," post-mortem elements if relevant, and the personal/reflective texture that the shorter platforms can't carry.

**Critical:** the Newsletter should include something the other two don't — usually (a) the full mechanism, (b) invalidation conditions, (c) a "what I got wrong last time" element if applicable.

**Output:** 1,000–2,000 word essay. Section headers welcome. Voice per `platform-tone.md` → Newsletter.

## The Annoyance Test (Hard Gate)

After drafting all active-platform posts, run this test before presenting to Peter:

> **"If someone followed Peter on all three active platforms, would they be annoyed seeing substantially the same content reworded?"**

- **If yes** → you're reformatting, not rethinking. Go back to the relevant step and choose a different angle.
- **If no** → the content engine did its job. Three pieces that each think about the topic differently.

### What "different" actually means

Three posts are sufficiently different when at least three of these differ meaningfully:
1. **Opening angle** (which facet of the topic the post leads with).
2. **Hook type** (from `hooks.md` — ideally different family per platform).
3. **Structure** (thread vs. long-form post vs. essay with headers).
4. **Depth** (surface insight / walk-through / full mechanism + invalidation).
5. **Voice energy** (per `platform-tone.md` — punchy / institutional-warm / reflective).

If only the **length** and the **line breaks** differ, the annoyance test fails.

## What Changes Per Platform

| Dimension | X | LinkedIn | Newsletter |
|-----------|---|----------|------------|
| **Angle** | Sharpest single facet | Different facet — often a framework or walk-through framing | Full picture with context |
| **Hook family** | Data Lead, Non-Obvious Angle, Proof, Collision | Framework, Walk-Through, Non-Obvious Angle, Collision | Walk-Through, Honest Correction, Collision |
| **Tone energy** | Direct, punchy, light humor allowed | Professional and warm, narrative arcs | Reflective, personal, honest about uncertainty |
| **Format** | Single tweet or 5–12 tweet thread | 1,300–2,000 char LinkedIn post | 1,000–2,000 word essay |
| **Length** | 280 chars (single) to ~3,000 chars (thread) | ~1,300–2,000 chars | ~1,000–2,000 words |
| **Depth** | Surface insight + core claim | Walk-through of reasoning + one counterargument | Full mechanism + counterargument + invalidation + post-mortem if relevant |
| **Close** | Concrete takeaway or genuine question | Question inviting perspective | Personal note or reply-inviting question |

## Batch Workflow

Peter's default batching pattern:

- **Sunday (30–60 min):** Pick 2–3 topics for the week. Run the full repurposing chain for each. Produces 6–9 posts (X + LinkedIn + Newsletter per topic).
- **Monday–Friday (10–15 min/day):** Review engagement, reply to meaningful comments in the first hour of posting, note hook/angle performance.
- **Friday (20 min):** Weekly review. Which hooks/angles performed? Update `hooks.md` with winners. Note anything to add to `platform-tone.md` based on what sounded right.

---

## When to Break the Chain

The chain is the default. Exceptions:

- **Reaction post to a breaking event.** Sometimes the X post needs to ship within an hour. Post X alone; decide later whether to expand to LinkedIn/Newsletter.
- **Newsletter-native content.** Some ideas (post-mortems, long-form deep dives) belong only in the Newsletter. Don't force them into X/LinkedIn if the compression destroys the value.
- **X-native formats.** Quote-tweet analysis often doesn't carry to LinkedIn — it loses context. Skip the LinkedIn step if the post is fundamentally X-shaped.

When breaking the chain, note it explicitly in the output. Don't silently skip a platform.
